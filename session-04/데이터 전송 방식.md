## ✅ 데이터 전송 방식 (Request)

- **HTTP 메시지를 통해 서버에 데이터를 전송(요청)**
- start Line : (HTTP 메서드) (Path) [? Query] (HTTP-version)
- Headers : HTTP 메시지 전송을 위한 필요 정보
- Body : 서버로 보낼 실제 데이터

```
[ HTTP 메시지 전체 구조 ex]

[start Line] ---> GET /users/10?active=true HTTP/1.1
[Headers] ------> Content-Type: application/json

[Body] ---------> { "name": "Kim" }
```

## ✅ Path, Query, Body(JSON)의 역할

- Path : 얻고자 하는 자원의 경로 (무엇을?) [필수]
- Query : 속성=값 으로 얻고자 하는 자원의 조건 명시 (어떤 조건?) [선택]
- Body : 서버에 전달할 실제 데이터 (무슨 데이터를 보낼까?) [선택]

## ❓ 참고

- Body 에는 JSON 말고도 여러 데이터들이 들어갈 수 있음 (이미지, 영상, XML, Text 등)
- JSON은 현재 많이 사용하는 데이터 구조
- 민감한 데이터는 Body에 작성 + HTTPS
- Body = 보통 POST, PUT, PATCH >>> 데이터 삽입 or 수정시 사용

## ❗ Body 사용시 주의할 점

- Body의 내용을 Query에 작성시 URL에 그대로 노출되면 보안상 문제가 생길 수 있음
- Body에 작성시 URL에 보이지 않으며 노출 위험성 낮아짐
- 단, HTTP가 아닌 HTTPS 프로토콜을 사용하여야 함
- 데이터는 Body (REST 설계 원칙)
- HTTP = 평문 전송 / HTTPS = TLS 암호화됨

| 구분 | HTTP | HTTPS |
| --- | --- | --- |
| 전송 방식 | 평문 (그대로 보임) | 암호화됨 |
| 도청 위험 | 매우 높음 | 거의 없음 |
| 데이터 보호 | ❌ 없음 | ✅ 있음 |
| 비밀번호 안전성 | ❌ 위험 | ✅ 안전 |
