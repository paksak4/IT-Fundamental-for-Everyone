# HTTP 메서드 속성

## 1️⃣ 안전성 (Safe)

- 요청을 보내도 **서버의 상태가 변경되지 않는 성질**
    - 즉, **읽기 전용** 요청이라 할 수 있음

### 안전한 메서드

- `GET`
- `POST`

### 특징

- 데이터 조회만 수행
- 서버 데이터 변경 X
- 여러 번 요청해도 문제 X

## 2️⃣ 멱등성 (Idempotent)

- 같은 요청을 여러 번 보내도 **결과가 동일한 성질**
- 응답이 같아야 하는 게 아니라, **서버 상태**가 같아야 함

### 멱등한 메서드

- `GET`
- `POST`
- `DELETE`
- `HEAD`

---

- **예시**
    
    ```
    PUT /users/1
    ```
    
    - 같은 데이터를 10번 보내도 결과 동일
    
    ```
    DELETE /users/1
    ```
    
    - 1번 삭제하든 n번 삭제하든 결과는 `삭제됨` 임

---

### 멱등하지 않은 메서드

- `POST`
- `PATCH`  (상황에 따라 다름)

---

- **예시**
    
    ```
    POST /users
    ```
    
    - 요청할 때마다 새로운 사용자 생성됨

---

## 3️⃣ 캐시 가능성 (Cacheable)

- 응답 결과를 저장해두고 **재사용할 수 있는 성질**

### 캐시 가능한 메서드

- `GET`
- 이론적으로 `POST`도 가능하지만 거의 사용X

### 동작 방식

1. 클라이언트가 요청
2. 서버가 응답 + 캐시 정보 전달
3. 이후 동일한 요청 발생 → 캐시 데이터 사용

### 장점

- 응답 속도 향상
- 서버 부하 감소
- 네트워크 비용 절감

## ▶️ 비교 정리
| 메서드 | 안정성 | 멱등성 | 캐시 |
| --- | --- | --- | --- |
| GET | O | O | O |
| POST | X | X | X (거의 사용 X) |
| PUT | X | O | X |
| PATCH | X | X (경우에 따라 다름) | X |
| DELETE | X | O | X |
