**URL 입력부터 서버 응답까지의 과정**

우리가 웹 브라우저에 URL을 입력하면 서버에서 데이터를 받아와 웹 페이지가 화면에 표시된다.  
그 과정에서는 여러 단계의 네트워크 통신이 이루어진다.

---

# 1️⃣ URI와 URL

## URI (Uniform Resource Identifier)

인터넷에 있는 자원을 식별하기 위한 **주소 체계**

URI는 크게 두 가지로 나뉜다.

- **URL (Uniform Resource Locator)** : 자원의 위치
- **URN (Uniform Resource Name)** : 자원의 이름

일반적으로 우리가 사용하는 웹 주소는 **URL**이다.

### 예시


https://www.google.com/search?q=java


### URL 구조


scheme : https  
host : www.google.com  
path : /search  
query : q=java   


---

# 2️⃣ URL 입력

사용자가 브라우저 주소창에 URL을 입력한다.


https://www.google.com


브라우저는 URL을 분석하여 다음 정보를 확인한다.

- 어떤 **프로토콜**인지
- 어떤 **서버**인지
- 어떤 **경로**인지

---

# 3️⃣ DNS 조회

브라우저는 **DNS 서버에 요청**을 보내 도메인 이름을 **IP 주소로 변환**한다.

### 예시


www.google.com
 → 142.250.206.14


DNS가 없다면 사용자는 다음과 같이 접속해야 한다.


http://142.250.206.14


사람이 기억하기 어렵기 때문에 **DNS 시스템이 필요하다.**

---

# 4️⃣ TCP 연결 생성 (3-Way Handshake)

IP 주소를 알게 되면 브라우저는 서버와 **TCP 연결** 을 생성한다.

### 과정


클라이언트 → SYN → 서버  
서버 → SYN + ACK → 클라이언트  
클라이언트 → ACK → 서버  


### 용어

- **SYN (Synchronize)**  
  연결을 시작할 때 보내는 신호

- **ACK (Acknowledgment)**  
  메시지를 잘 받았다는 확인 신호

이 과정이 완료되면 **클라이언트와 서버 간 연결이 생성된다.**

---

# 5️⃣ HTTP 요청 전송

TCP 연결이 생성되면 브라우저는 서버에 **HTTP 요청(Request)** 을 보낸다.

### 예시


GET / HTTP/1.1
Host: www.google.com


의미


www.google.com
 서버의 / 경로 데이터를 요청


---

# 6️⃣ 서버 응답

서버는 요청을 처리한 후 **HTTP 응답(Response)**을 보낸다.

### 예시


HTTP/1.1 200 OK
Content-Type: text/html


서버는 HTML 데이터를 함께 전달한다.

---

# 7️⃣ 브라우저 렌더링

브라우저는 서버로부터 받은 데이터를 해석하여 웹 페이지를 화면에 표시한다.

렌더링 과정에서 다음 리소스를 추가로 요청할 수 있다.

- HTML
- CSS
- JavaScript
- Image

이 리소스들을 처리하여 **최종 웹 페이지가 화면에 표시된다.**

---

# 🔄 전체 흐름 정리


1.사용자가 URL 입력

2.DNS 조회 (도메인 → IP 주소)

3.TCP 연결 생성 (3-Way Handshake)

4.HTTP 요청 전송

5.서버가 HTTP 응답 반환

6.브라우저 렌더링
