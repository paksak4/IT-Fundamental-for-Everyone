## 클라이언트 - 서버 구조

클라이언트는 요청(Request)을 보내고 서버는 요청(Response)을 처리한 후 응답을 반환한다. 

`
클라이언트 --- Request ---> 서버
			
      (서버에서 요청 처리)

클라이언트 <-- Response --- 서버
`

집에 있는 컴퓨터로 크롬을 통해 구글에 접속한다고 예시를 들면, 집에 있는 컴퓨터에서 실행되는 크롬 웹 브라우저는 클라이언트이고, 구글 웹사이트를 제공하는 구글의 웹 서버가 서버에 해당한다.

---

## 역할 분리 

위와 같은 구조의 클라이언트와 서버는 각자의 역할이 명확하게 나뉜다. 

#### 클라이언트
- 사용자의 입력 받기
- 서버에 요청 보내기
- 서버 응답을 화면에 표시
- 사용자 인터페이스 제공
- 예시: 웹 브라우저(크롬, 사파리), 모바일 웹, PC 프로그램

#### 서버
- 클라이언트로부터 온 요청 처리
- 데이터 저장 및 관리
- 비즈니스 로직 연산 수행
- 클라이언트에 응답 생성 및 전송
- 예시: 웹 서버, API 서버, 데이터베이스 서버

즉, 클라이언트는 사용자와 직접 상호작용하는 역할에, 서버는 요청을 처리하고 데이터를 제공하는 역할에 집중한다.

---

## 이러한 역할 분리 구조의 대표적인 장점 

- 클라이언트와 서버를 따로 개발하여 효율적으로 독립적인 개발이 가능하다.
- 역할이 명확하기 때문에 유지보수가 수월하며, 서버만 확장하는 등 확장성에도 용이하다.

<br>

이러한 구조는 웹의 기본 아키텍처로 사용되며, HTTP 통신 역시 클라이언트-서버 구조를 기반으로 동작한다.
